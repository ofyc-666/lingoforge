# LingoForge MVP 验收标准

## 验收目标

MVP 验收目标是证明一个最小自适应训练闭环成立：

**真题为源，Skill 为教学能力，Agent 负责调度，确定性程序负责判分，未见真题负责验收。**

MVP 不验收长期提分，不宣称训练已经产生因果效果。

## 1. 范围验收

MVP 验收通过需满足：

- 只覆盖 CET-6 词汇与阅读；
- 完成第一次完整主线学习；
- 至少完成一个机场购票副线任务；
- 副线信号进入候选池或待验证标记；
- 第二次自适应主线计划基于更新后的画像和副线信号生成；
- 至少完成一个体现自适应变化的短训练任务；
- 短训练任务后包含短隔离检测；
- 不实现听力、口语、完整写作教学、其他考试、真正 3D、多场景、社交、支付、多 Agent 或 Agentic RL。

验收不通过示例：

- 只完成第一次完整主线学习，未生成第二次自适应计划或未完成对应短训练任务；
- 副线功能扩大为多个场景或复杂养成；
- 用普通 AI 生成题替代 CET-6 Skill 约束；
- 用模型自述代替确定性判分。

## 2. 主线闭环验收

主线闭环验收通过需满足：

- 用户完成首次目标收集；
- 用户完成短诊断；
- 系统保存短诊断原始证据；
- 系统形成初始画像；
- 程序筛选候选词；
- Agent 在候选范围内调度第一次主线学习；
- 用户完成低压力学习和真题风格迁移训练；
- 程序完成客观判分；
- Agent 基于实际证据提出结构化画像更新建议；
- 程序校验并决定是否应用画像更新；
- 用户完成机场购票副线任务；
- 副线信号进入主线候选池或触发待验证标记；
- 第二次自适应主线计划基于更新后的画像和副线信号发生可解释变化；
- 用户至少完成一个体现该变化的短训练任务；
- 短训练任务后进入短隔离检测。

第二次主线计划的自适应变化，必须体现在以下至少一项：

- Skill 选择变化；
- 目标能力变化；
- 难度参数变化；
- 题型或任务类型变化；
- 提示策略变化。

仅文章主题变化不算自适应成立。

第二次主线不要求再次完成一套完整学习闭环，也不要求在短训练任务后再次完成完整画像更新。若短训练任务包含客观题，仍须由确定性程序判分并保存证据。

## 3. 真题证据与 Skill 验收

MVP 的 4 类 CET-6 阅读 Skill 必须存在最小可运行、可测试版本：

1. 词汇语境识别 Skill；
2. 长难句与逻辑关系 Skill；
3. 同义替换与定位 Skill；
4. 干扰项判断 Skill。

每类 Skill 必须明确：

- 目标能力；
- 适用条件；
- 输入与输出结构；
- 可调难度参数；
- 基于真题规律的训练或题目生成规则；
- 答案依据；
- 常见错误类型；
- 最小质量校验规则。

Skill 验收通过需满足：

- Skill 不是独立 Agent；
- Skill 不是泛化提示词；
- Skill 可由同一个学习 Agent 调度；
- Skill 有标识与版本；
- Skill 的真题规律来源、适用范围和版本属于 Skill 元数据；
- 运行时能引用 Skill 版本；
- 生成训练内容时能声明质量校验要求；
- 程序或独立校验环节能产出质量校验结果。

验收不通过示例：

- 只写“生成一篇六级风格文章”；
- 无法说明题目考察哪个能力；
- 标准答案无法回指材料；
- 干扰项没有错误依据；
- 把模型自称“质量通过”当成校验结果。

## 4. Skill 契约验收

MVP 采用“统一公共契约 + 按任务类型区分的载荷”。

公共输入语义验收：

- 包含 Skill 标识与版本；
- 包含目标能力；
- 包含当前任务场景；
- 包含与目标能力相关的用户画像切片；
- 包含最近相关证据摘要；
- 包含程序筛选的候选词；
- 包含难度与提示参数。

公共输出语义验收：

- 包含本轮训练目标；
- 包含材料或任务规格；
- 包含生成内容；
- 包含本任务预期观察的能力与证据；
- 包含 Skill 版本与生成依据；
- 包含需要执行的质量校验要求。

客观训练题或隔离检测题额外验收：

- 包含标准答案；
- 包含可回指材料的答案依据；
- 包含干扰项错误依据；
- 包含可能暴露的错误类型。

低压力学习、长难句拆解或补救讲解验收：

- 不强制包含标准答案；
- 不强制包含干扰项依据；
- 必须包含适合任务类型的分层提示、讲解结构或练习要求；
- 必须说明可观察的能力或证据。

当前验收不要求锁定最终 JSON Schema、字段类型、接口设计或具体实现。

## 5. 用户画像验收

用户画像验收通过需满足：

- 使用 4 类 Skill 对齐的最小能力维度；
- 横向记录提示依赖程度、常见错误类型、最近证据来源和当前置信度；
- 横向记录尽量与具体能力维度关联；
- 原始学习行为和答题证据与派生画像分开保存；
- 画像更新建议包含能力维度、变化方向、理由和证据引用；
- Agent 只能提出结构化更新建议；
- 程序负责校验并决定是否应用；
- 副线信号不直接修改正式能力画像。

验收不通过示例：

- 画像只有“阅读弱”“词汇弱”等泛化标签；
- 画像更新没有证据引用；
- Agent 直接修改掌握度；
- 副线点击行为直接改变正式画像；
- 原始证据被画像覆盖或替代。

当前验收不要求确定数据库结构、数值评分公式、权重、阈值或单次更新幅度。

## 6. 判分与证据验收

确定性程序必须负责：

- 客观题判分；
- 计时；
- 保存题目版本；
- 保存 Skill 版本；
- 保存用户答案；
- 保存用时；
- 保存提示使用；
- 保存交互行为；
- 保存质量校验结果；
- 校验画像更新建议。

验收通过需证明：

- 客观题结果不是由 Agent 自由判定；
- 原始证据可追溯；
- 派生画像能回指原始证据；
- 质量校验结果来自程序或独立校验环节；
- Agent 的解释不能覆盖原始判分。

## 7. 隔离检测验收

短隔离检测验收通过需满足：

- 出现在第二次自适应短训练任务之后；
- 来自预先保留的未见题集或合法样题风格保留题；
- 训练材料生成、补救练习和 Agent 调度过程不可见；
- 不能由 Agent 根据本轮用户表现临时生成后再自行验证；
- 至少覆盖第二次主线重点调度的 1 个能力维度；
- 不要求覆盖全部 4 类 Skill；
- 检测期间不提供提示、分层讲解、答案反馈或动态降难度；
- 提交后统一显示结果与解释；
- 由确定性程序判分；
- 保存题目版本、用户答案、用时和结果。

隔离检测形式可为：

- 一篇短材料配 2 到 3 道客观题；
- 3 道左右相互独立的客观小题。

隔离检测验收只证明：

- 隔离题与训练内容确实分离；
- 系统能够获取未见任务上的独立表现证据；
- 该证据能够进入后续画像与调度闭环。

隔离检测不得用于宣称：

- 已经证明长期提分；
- 已经证明训练具有因果效果；
- 用户 CET-6 总成绩必然提高。

如首次诊断中存在同一能力维度的可比证据，可以展示前后表现差异。如题目难度或测量方式不可比，则不得直接宣称能力提高。

## 8. 副线验收

副线验收通过需满足：

- 采用 2D 或 2.5D 赛博小镇；
- MVP 只包含一个机场场景；
- MVP 只包含一个购买机票任务；
- 用户可在任务中接触机场环境词汇和主线薄弱词；
- 副线数据与主线学习证据分开保存；
- 副线信号标记来源场景、相关词汇或表达、任务上下文和发生时间；
- 副线信号可进入主线候选池或触发“待正式验证”标记；
- 副线信号不直接修改 4 类正式能力画像；
- 副线数据不得作为隔离检测证据；
- 副线不得与隔离题集产生内容泄漏。

副线中的客观任务结果，例如是否选对航班，只能证明该副线任务是否完成，不能直接推导为 CET-6 正式能力提升。

同一词汇在副线中多次点击、提示依赖或识别失败，可以增强其待验证优先级，但具体累积规则、衰减规则和权重留到架构设计阶段确定。

## 9. 不可接受的花架子实现

以下情况应判定为未通过 MVP 规格验收：

- 主线只是用户画像驱动生成内容，没有真题 Skill 约束；
- Skill 只是 4 段泛化提示词；
- Agent 既生成题又自判分又自称用户进步；
- 第二次计划只换了文章主题；
- 没有保存原始证据；
- 没有区分原始证据和派生画像；
- 副线点击直接改变掌握度；
- 隔离题由 Agent 临时生成并自证；
- 使用隔离检测宣称长期提分；
- 为了展示效果擅自加入非 MVP 功能。

## 10. MVP 完成判定

MVP 规格层面完成需满足：

- `docs/MVP_SCOPE.md` 明确范围与不做事项；
- `docs/USER_FLOW.md` 明确完整用户流程；
- `docs/AGENT_RESPONSIBILITIES.md` 明确 Agent、Skill、Workflow 和确定性程序边界；
- `docs/ACCEPTANCE_CRITERIA.md` 明确可测试验收标准。

后续进入架构设计或代码实现前，需要人工审核并确认这些规格。

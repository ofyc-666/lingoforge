# LingoForge MVP 范围

## 当前阶段

LingoForge 当前处于 MVP 产品规格梳理阶段。本阶段只明确产品范围、用户流程、Agent 职责边界、Skill 语义边界、主副线数据关系和可验收标准。

本阶段不做：

- 应用代码实现；
- 前端或后端框架初始化；
- 最终技术栈选择；
- 详细视觉稿设计；
- 数据库结构、接口字段、权重、阈值或评分公式锁定；
- 完整架构实现。

## 产品定位

LingoForge 是面向英语考试与长期英语学习的自适应学习 Agent。MVP 聚焦 CET-6 词汇与阅读，目标不是做通用聊天式英语助手，也不是做完整游戏化学习平台，而是验证一个最小闭环：

**真题为源，Skill 为教学能力，Agent 负责调度，确定性程序负责判分，未见真题负责验收。**

## 核心价值假设

用户的真实痛点不是缺少英语内容，而是：

- 不知道自己阅读题错在哪里；
- 不知道下一次最该练什么；
- 背词、刷题和错因分析之间割裂；
- AI 生成内容难以证明是否真接近考试要求；
- 模型自称用户进步，但缺少独立证据。

MVP 需要证明：系统能够基于 CET-6 真题或合法考试样题炼化出的结构化 Skill，结合用户画像与本轮表现，调度下一次训练，并用确定性判分和短隔离检测获得可追溯证据。

MVP 不证明：

- 长期提分；
- 训练与成绩提升之间的因果效果；
- 多周复习策略有效；
- 用户长期留存；
- 对其他考试或其他英语能力的泛化效果。

## MVP 最小闭环

MVP 采用以下最小闭环：

1. 首次对话收集用户目标与学习约束；
2. 完成短诊断；
3. 形成初始用户画像；
4. 完成第一次完整主线学习；
5. 完成第一次主线后的画像更新；
6. 完成一次机场购票副线任务；
7. 副线信号进入候选池或待验证标记；
8. 根据更新后的画像和副线信号生成第二次自适应主线计划；
9. 至少完成一个体现自适应变化的短训练任务；
10. 进入短隔离检测。

第二次主线不要求再次完成一套完整学习闭环。MVP 对第二次主线的要求仅为：

- 根据更新后的画像和副线信号重新生成计划；
- 明确解释计划变化依据；
- Skill 选择、目标能力、难度、题型或提示策略至少一项发生有证据的变化；
- 至少完成一个体现该变化的短训练任务；
- 随后进入短隔离检测。

第二次主线计划的自适应变化不能只是文章主题变化，必须体现在以下至少一类变化中：

- Skill 选择；
- 目标能力；
- 难度参数；
- 题型或任务类型；
- 提示策略。

## 主线范围

MVP 主线聚焦 CET-6 词汇与阅读。

主线包括：

- 首次目标收集；
- 短诊断；
- 初始用户画像；
- CET-6 阅读 Skill 驱动的训练；
- 包含目标词汇的低压力阅读学习；
- 由 CET-6 Skill 约束的真题风格迁移训练；
- 客观题确定性判分；
- 错因分析；
- 结构化画像更新建议；
- 程序校验后的画像更新；
- 第二次自适应主线计划；
- 至少一个体现自适应变化的短训练任务；
- 短隔离检测。

主线不包括：

- 听力训练；
- 口语训练；
- 完整写作教学；
- 雅思、托福等其他考试；
- 直接复制真题文本；
- 仅依赖通用大模型声称内容“像六级”；
- Agent 自行修改核心成绩或掌握度。

## 真题与 Skill 边界

CET-6 真题或合法考试样题，是主线能力标签、题型规律、语言难度、同义替换方式、干扰项模式和评价标准的主要依据。

系统将这些规律炼化为结构化教学 Skill：

- 不直接复制真题文本；
- 不把真题原文暴露给训练生成流程；
- 不把 Skill 实现为泛化提示词；
- 不把 Skill 实现为多个独立 Agent；
- 由同一个学习 Agent 调度这些 Skill。

MVP 采用 4 类 CET-6 阅读 Skill 作为最小 Skill 范围，顺序按用户实际阅读和解题过程排列：

1. 词汇语境识别 Skill；
2. 长难句与逻辑关系 Skill；
3. 同义替换与定位 Skill；
4. 干扰项判断 Skill。

这 4 个能力维度只是 MVP 用于证明自适应闭环的最小画像模型，不代表未来完整产品的能力体系永久限定为 4 类。

每类 Skill 在 MVP 中只要求实现一个可运行、可测试的最小版本，不需要一次覆盖所有 CET-6 阅读子题型。

每类 Skill 至少需要明确：

- 目标能力；
- 适用条件；
- 输入与输出结构；
- 可调难度参数；
- 基于真题规律的训练或题目生成规则；
- 答案依据；
- 常见错误类型；
- 最小质量校验规则。

## 用户画像范围

MVP 采用“4 类 Skill 对齐 + 提示依赖横向记录”的最小画像模型。

主能力维度：

- 词汇语境识别；
- 长难句与逻辑关系；
- 同义替换与定位；
- 干扰项判断。

横向记录：

- 提示依赖程度；
- 常见错误类型；
- 最近证据来源；
- 当前置信度。

提示依赖程度、常见错误类型、证据来源和当前置信度，应尽量与具体能力维度关联。例如，用户可能在长难句理解上高度依赖提示，但在词汇语境识别上并不依赖。

原始学习行为和答题证据必须与派生画像分开保存。画像是根据证据形成的可更新判断，不能替代原始记录。

当前规格阶段只确定画像语义和更新原则，不确定具体数据库结构、数值评分公式、权重、阈值或单次更新幅度。

## 副线范围

MVP 副线采用 2D 或 2.5D 赛博小镇，不做真正 3D 开放世界。

MVP 只实现一个机场场景和一个“购买机票”任务。

机场任务可以包括：

- 阅读英文航班信息牌；
- 理解 departure、arrival、gate、round trip 等环境词汇；
- 根据 NPC 的英文要求选择合适航班；
- 填写简化购票信息；
- 完成简短英文对话或选择任务；
- 在陌生语境中再次识别主线接触过的词。

副线用于提高英语暴露量和学习趣味，不直接代表考试能力提高。

副线只产生曝光与待验证信号，不直接更新正式画像。

## 副线数据边界

副线可记录：

- 用户遇到的环境词、目标词或短语；
- 是否点击查看释义或提示；
- 是否在购票选择、表单填写、简短对话中使用或识别成功；
- 相关任务上下文；
- 可客观判断的任务结果，例如是否选对航班；
- 是否应提高相关词汇或表达进入下一次主线候选池的优先级。

副线不可做：

- 直接更新 4 类正式能力维度；
- 直接认定某词已掌握或未掌握；
- 作为隔离检测证据；
- 让 Agent 因副线表现直接修改核心成绩或掌握度；
- 与隔离题集产生内容泄漏。

副线中的客观任务结果只能证明该副线任务是否完成，不能直接推导为 CET-6 正式能力提升。

同一词汇在副线中多次点击、提示依赖或识别失败，可以增强其待验证优先级，但具体累积规则、衰减规则和权重留到架构设计阶段确定。

## 短隔离检测范围

MVP 在第二次自适应短训练任务后加入短隔离检测，作为正式验收能力证据的最小方案。

隔离检测题应从预先保留的未见题集或合法样题风格保留题中取得。该题集在训练材料生成、补救练习和 Agent 调度过程中不可见，不能由 Agent 根据本轮用户表现临时生成后再自行验证。

MVP 中的短隔离检测建议采用以下任一形式：

- 一篇短材料配 2 到 3 道客观题；
- 3 道左右相互独立的客观小题。

隔离检测只需覆盖第二次主线重点调度的至少 1 个能力维度，不要求覆盖全部 4 类 Skill。

检测期间不提供提示、分层讲解、答案反馈或动态降难度；提交后再统一显示结果与解释。

## 当前默认假设

以下是当前默认假设，用于形成可验收 MVP 规格；后续架构设计阶段可以在不破坏 MVP 目标的前提下调整实现细节：

- 短诊断覆盖至少 4 类能力维度中的部分关键能力，但不要求完整考试化；
- 第二次主线计划必须能解释其变化依据，并可使用更新后的画像和副线待验证信号；
- 第二次主线不要求再次完成完整学习闭环；
- 每类 Skill 在 MVP 中只做最小可运行版本；
- 隔离检测只覆盖第二次主线重点能力，不覆盖全部能力；
- 副线机场任务只服务于曝光、迁移和待验证信号，不承载正式评估。

## 明确不做

MVP 暂不实现：

- 听力；
- 口语；
- 完整写作教学；
- 雅思、托福等其他考试；
- 真正 3D 开放世界；
- 多个完整虚拟场景；
- 复杂角色养成与经济系统；
- 视频生成和自动字幕；
- 社交与排行榜；
- 支付与商业订阅；
- 多 Agent 协作；
- Agentic RL；
- 长期提分证明；
- 完整 CET-6 全题型覆盖；
- 详细视觉设计。

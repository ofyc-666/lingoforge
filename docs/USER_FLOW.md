# LingoForge MVP 用户流程

## 流程总览

MVP 用户流程围绕一个最小可验收闭环展开：

首次目标收集 -> 短诊断 -> 初始画像 -> 第一次完整主线学习 -> 画像更新 -> 机场副线任务 -> 副线信号进入候选池或待验证标记 -> 第二次自适应主线计划 -> 至少完成一个体现自适应变化的短训练任务 -> 短隔离检测。

主线负责正式训练与证据积累。副线负责低压力英语暴露和趣味迁移。隔离检测负责提供未见任务上的独立表现证据。

## 1. 首次目标收集

用户首次使用时，Agent 通过对话了解：

- 用户准备的考试；
- 距离考试还有多少天；
- 目标成绩；
- 每天可学习时间；
- 用户自认为的薄弱项；
- 用户感兴趣的内容主题。

MVP 默认考试为 CET-6。若用户表达其他考试目标，MVP 只记录为非当前支持范围，不进入其他考试训练。

该阶段输出：

- 用户目标摘要；
- 学习约束；
- 兴趣主题；
- 用户自评弱项；
- 后续短诊断的初始参考信息。

## 2. 短诊断

用户完成短诊断。短诊断用于建立初始证据，不用于证明长期能力。

确定性程序负责：

- 计时；
- 展示题目；
- 收集用户答案；
- 客观题判分；
- 保存题目版本、用户答案、用时、提示使用情况和结果。

Agent 负责：

- 结合用户目标和诊断结果形成初始画像；
- 标注画像判断依据；
- 不直接修改核心成绩或掌握度。

短诊断应尽量覆盖 MVP 的 4 类能力维度：

- 词汇语境识别；
- 长难句与逻辑关系；
- 同义替换与定位；
- 干扰项判断。

当前默认假设：短诊断不要求完整覆盖全部 CET-6 阅读子题型，只需为第一次主线调度提供足够初始证据。

## 3. 初始用户画像

系统根据短诊断和首次对话形成初始用户画像。

画像采用“4 类 Skill 对齐 + 提示依赖横向记录”的最小模型。

主能力维度：

- 词汇语境识别；
- 长难句与逻辑关系；
- 同义替换与定位；
- 干扰项判断。

每个能力维度尽量关联：

- 提示依赖程度；
- 常见错误类型；
- 最近证据来源；
- 当前置信度。

原始诊断证据与派生画像分开保存。画像是可更新判断，不替代原始记录。

## 4. 第一次主线学习

第一次主线学习用于启动正式训练闭环。

### 4.1 候选词筛选

确定性程序先根据以下信息筛选候选词：

- CET-6 考试词库；
- 复习日期；
- 历史错误；
- 当前用户画像；
- 可用训练范围。

Agent 只能在程序给出的候选范围内决定今日新词与复习词比例，不能绕过词库和复习规则自由选词。

### 4.2 Agent 调度

Agent 读取用户画像和候选词，决定：

- 本轮训练目标；
- 调用哪些 CET-6 阅读 Skill；
- 今日新词与复习词比例；
- 材料主题；
- 文章长度；
- 语言难度；
- 提示等级；
- 是否进入低压力学习、迁移训练或补救练习。

Agent 的调度必须能回指到画像、历史证据或本轮表现，不能只用泛化理由。

### 4.3 低压力阅读学习

系统生成一篇包含目标词汇、风格接近 CET-6 的连贯阅读材料，用于低压力学习。

用户可进行：

- 点击单词查看语境释义；
- 点击句子获取分层提示；
- 查看长难句拆解；
- 在不立即考试化的语境中理解目标词和句子。

低压力学习可以帮助用户建立语境理解，但不能单独作为正式能力提升证据。

### 4.4 真题风格迁移训练

低压力学习后，用户必须进入由 CET-6 Skill 约束的真题风格迁移训练。

迁移训练必须体现：

- 使用的 Skill 标识和版本；
- 目标能力；
- 题目或任务规格；
- 预期观察的能力和证据；
- 质量校验要求。

若任务是客观训练题，还必须具备：

- 标准答案；
- 可回指材料的答案依据；
- 干扰项错误依据；
- 可能暴露的错误类型。

### 4.5 判分与行为记录

确定性程序负责：

- 客观题判分；
- 保存用户答案；
- 保存用时；
- 保存提示使用；
- 保存点击和交互行为；
- 保存题目版本和生成版本；
- 保存质量校验结果。

不能把模型自称“已检查通过”当作质量校验结果。

## 5. 第一次主线后的画像更新

用户完成第一次主线后：

1. 程序完成客观判分并保存原始证据；
2. Agent 根据实际证据分析错误原因；
3. Agent 提出结构化画像更新建议；
4. 程序校验建议是否合法；
5. 程序决定是否应用到派生画像。

Agent 的画像更新建议至少包含：

- 能力维度；
- 变化方向；
- 理由；
- 证据引用。

Agent 不能直接修改正式能力画像、核心成绩或掌握度。

## 6. 机场购票副线任务

MVP 副线只实现一个机场场景和一个购买机票任务。

用户在副线中可以：

- 移动虚拟形象；
- 阅读英文航班信息牌；
- 理解机场环境词汇；
- 根据 NPC 英文要求选择合适航班；
- 填写简化购票信息；
- 完成简短英文对话或选择任务；
- 在陌生语境中再次遇到主线薄弱词或表达。

副线任务结果可以说明用户是否完成该副线任务，例如是否选对航班，但不能直接推导为 CET-6 正式能力提升。

## 7. 副线信号回流

副线信号必须与主线学习证据分开保存，并标记：

- 来源场景；
- 相关词汇或表达；
- 任务上下文；
- 发生时间；
- 用户点击、提示依赖或识别情况；
- 客观任务结果。

副线信号可以：

- 提高相关词汇或表达进入下一次主线候选池的优先级；
- 触发“待正式验证”标记；
- 提醒 Agent 在后续主线中安排正式验证。

副线信号不能：

- 直接修改 4 类正式能力画像；
- 直接认定词汇掌握或未掌握；
- 作为隔离检测证据；
- 与隔离题集产生内容泄漏。

只有主线训练或隔离检测中的正式证据，经过程序校验后，才能改变正式能力画像。

## 8. 第二次自适应主线计划

第二次自适应主线计划用于验证系统是否能基于更新后的画像和副线待验证信号调整训练，而不是要求用户再次完成一整套主线学习闭环。

第二次计划必须：

- 根据更新后的画像和副线信号重新生成；
- 明确解释计划变化依据；
- 在 Skill 选择、目标能力、难度、题型或提示策略中至少一项发生有证据的变化；
- 说明哪些副线词汇或表达进入候选池，哪些只被标记为待正式验证；
- 继续受程序候选词筛选和 Skill 约束。

仅改变文章主题，不算自适应成立。

## 9. 第二次短训练任务

用户至少完成一个体现第二次计划变化的短训练任务。

该短训练任务必须：

- 对应第二次计划中的自适应变化；
- 使用被选中的 Skill 或任务策略；
- 能产生可记录的学习证据；
- 若包含客观题，仍由确定性程序判分并保存证据。

MVP 不要求第二次主线再次完成完整学习闭环，也不要求在短训练任务后再次完成完整画像更新。若短训练任务证据用于画像更新，仍必须遵循 Agent 提出结构化建议、程序校验后应用的原则。

## 10. 短隔离检测

第二次短训练任务后加入短隔离检测。

隔离检测题来自预先保留的未见题集或合法样题风格保留题。该题集在训练材料生成、补救练习和 Agent 调度过程中不可见。

隔离检测可采用：

- 一篇短材料配 2 到 3 道客观题；
- 3 道左右相互独立的客观小题。

检测要求：

- 至少覆盖第二次主线重点调度的 1 个能力维度；
- 不要求覆盖全部 4 类 Skill；
- 检测期间不提供提示；
- 检测期间不提供分层讲解；
- 检测期间不提供答案反馈；
- 检测期间不动态降难度；
- 提交后统一显示结果与解释。

确定性程序负责：

- 判分；
- 保存题目版本；
- 保存用户答案；
- 保存用时；
- 保存结果。

隔离检测结果可以作为比普通训练更高置信度的画像证据，但仍须由程序按规则校验后更新画像。

如首次诊断中存在同一能力维度的可比证据，可以展示前后表现差异。如题目难度或测量方式不可比，则不得直接宣称能力提高。

## 11. MVP 流程完成条件

当以下事项全部完成时，MVP 用户流程闭环成立：

- 用户完成首次目标收集；
- 用户完成短诊断；
- 系统形成初始画像；
- 用户完成第一次完整主线学习；
- 系统完成第一次画像更新；
- 用户完成机场购票副线任务；
- 副线信号进入候选池或待验证标记，但没有修改正式画像。
- 第二次主线计划因画像和副线信号产生可解释变化；
- 用户完成至少一个体现自适应变化的短训练任务；
- 用户完成短隔离检测。
